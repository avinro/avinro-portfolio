> A decentralized domain marketplace designed to simplify bulk ENS domain registration for crypto-native users and domain collectors.

## Overview

DomainPlug was a Web3 marketplace concept built to solve a clear gap in the ENS experience: users could search and buy `.eth` domains, but there was no efficient way to browse, compare, and register multiple domains in a single transaction.

## Design Challenge

During the crypto boom of 2021–2023, ENS domain marketplaces were growing fast, but the registration experience was still designed around buying one domain at a time. The challenge was making bulk registration feel like a primary use case, not a hidden or advanced feature.

The experience needed to support:

- Bulk registration for up to 20 domains per transaction
- A mobile-first responsive interface
- A clean light-mode visual style, based on the client's requirements
- Fast execution using an existing component library

## Design Direction

The interface was built on top of **Taiga UI** and customized with the client's purple visual palette. This let the design process stay focused on the UX problem — making bulk ENS registration easy to understand, easy to compare, and easy to complete — rather than building component infrastructure from scratch.

## Result

The project delivered a complete high-fidelity responsive prototype. It remained a prototype after the client pivoted, but it validated bulk registration as a viable product direction for crypto-native users and domain collectors.

### What This Project Demonstrates

- **Pragmatic design systems:** Used Taiga UI strategically to move fast while keeping consistency.
- **Mobile-first UX:** Designed for phones first, then expanded the experience for desktop.
- **Feature clarity:** Made bulk registration visible, understandable, and easy to access.
- **Product thinking:** Prioritized user paths, information hierarchy, and purchase flow clarity.
